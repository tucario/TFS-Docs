كل حساب TranSFlator له **رمز API** واحد — وهو سلسلة عشوائية طويلة
تثبت هوية حسابك لدى نظام TranSFlator الخلفي لمكالمات واجهة برمجة تطبيقات الترجمة.

لا تحتاج عادةً إلى التفكير فيه. يستخدمه تطبيق سطح المكتب تلقائياً عند تسجيل الدخول. المرة الوحيدة التي تتفاعل فيها معه
مباشرة هي عندما تريد استدعاء واجهة برمجة تطبيقات الترجمة من التعليمات البرمجية الخاصة بك (مثل خط أنابيب CI/CD).

## أين يوجد

في اللوحة ← **الملف الشخصي ← رمز API**. يظهر الرمز مع زر **نسخ** وزر **إعادة توليد**.

## إعادة التوليد (التدوير)

انقر على **إعادة توليد** لإبطال الرمز الحالي وإصدار رمز جديد. استخدم هذا إذا كنت تعتقد أن الرمز قد تسرب — تم دفعه إلى مستودع عام، أو نُشر في دردشة، أو تُرك في ملف سجل — أو ببساطة
كجزء من جدول تدوير منتظم.

بعد إعادة التوليد، سيتلقى أي تطبيق سطح مكتب مسجل الدخول بالفعل بالرمز القديم خطأ `401 Unauthorized` في مكالمة API التالية وسيطلب منك تسجيل الدخول مرة أخرى.

## استخدامه مباشرة

أرسله كرمز Bearer في ترويسة `Authorization` عند
استدعاء واجهة برمجة تطبيقات الترجمة الخاصة بنا:

```bash
curl -X POST https://transflator-api.web.app/translate/batch \
  -H "Authorization: Bearer <YOUR_API_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{
    "strings": ["Account name", "Industry", "Annual revenue"],
    "target_language": "ar",
    "engine": "gemini"
  }'
```

الاستجابة هي مصفوفة JSON من النصوص المترجمة بنفس ترتيب الإدخال.
